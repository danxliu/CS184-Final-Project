01 — Accomplishments

CPU pipeline end-to-end through Phase 1

Two phases of the roadmap are complete and gated by regression tests: Phase 0 (infrastructure) and Phase 1 (discrete tangent-point energy with Barnes-Hut acceleration, including the gradient). Together they get us to "flow a mesh along $-\nabla \hat\Phi$ at $O(n \log n)$", which is the Week 1 target from the proposal.

Phase 0 — infrastructure

  • SoA mesh (MeshData: $V, F, N, L_0$) with an OpenMesh round-trip bridge for later topology-aware remeshing.
  • Finite-difference gradient checker with central differences — the validation gate every analytical gradient below has to pass before anything downstream depends on it.
  • Procedural test meshes: icosphere (any subdivision level) and torus with analytic normals.

Phase 1 — CPU TPE + BVH

  • Per-face geometry with closed-form Jacobians for centroid, unit normal, and area with respect to each incident vertex.
  • Brute-force TPE $\hat\Phi(x)$ and $\nabla \hat\Phi(x)$ at $\alpha = 6$ via midpoint quadrature over ordered face pairs.
  • SAH BVH over face AABBs with cluster aggregates (area, area-weighted centroid, unweighted normal-sum, radius) cached per node for the Barnes-Hut pass.
  • Block-cluster tree over BVH$\times$BVH with the RSu radius-form multipole acceptance criterion $\max(r_U, r_V) < \theta \, \|c_U - c_V\|$ at $\theta = 0.5$.
  • Barnes-Hut energy + gradient at $O(n \log n)$. Approximates admissible pairs by a 0th-order cluster proxy, sums the remaining leaf-leaf pairs exactly.
  • Viewer (NanoGUI + OpenGL) with Blinn-Phong shading and a frame-sequence playback mode (scrubber + Space/←/→ keys) used to debug the gradient flow below.
02 — Preliminary results

What we can measure now

Regression tests: 20/20 + 47/47 checks pass. test_phase0 exercises the OpenMesh round-trip, the normalization + $L_0$ cache, the FD checker on a known quadratic, and the test-mesh factories. test_phase1 adds 35 checks: FD gradient on face geometry and on brute-force TPE, scale-covariance $\hat\Phi(\lambda x) = \lambda^{4-\alpha}\hat\Phi(x)$ (zero relative error, as expected from the closed-form scaling), translation invariance, aggregate conservation in the BVH, exact exhaustive pair coverage in the BCT, and Barnes-Hut-vs-brute cross-checks.

Barnes-Hut accuracy. At $\theta = 0$ the Barnes-Hut energy matches brute force to $\sim 3 \times 10^{-15}$ relative error on a 192-face torus (pure floating-point summation-order noise). At $\theta = 0.5$ on icosphere$(3)$ (1280 faces) the relative error is $1.0 \times 10^{-3}$, matching the RSu $\S 5.2$ accuracy claim. A $\theta$-sweep on a thin torus gives monotone convergence: $\theta = 0.8 \to 6.9\%$, $\theta = 0.5 \to 4.4\%$, $\theta = 0.1 \to 1.1 \times 10^{-4}$.

Gradient flow demo. We descend $\hat\Phi$ with Armijo-backtracked $L^2$ gradient descent on an ellipsoid built by scaling icosphere$(2)$ along its principal axes by $(1.7, 0.75, 0.75)$. The uniform scaling mode is projected out of the gradient each step so descent moves shape, not size (TPE is scale-covariant with exponent $\lambda^{-2}$, so raw descent would inflate the mesh forever).

Figure. Phase 1.8 $L^2$ gradient flow on the ellipsoid, 95 iterations. The energy drops several-fold before stalling; the Armijo step size $\tau$ collapses to $\sim\!10^{-12}$ even though $\|\nabla\hat\Phi\|$ stays $O(10^{1\text{–}3})$. This is the expected $L^2$ failure mode (RSu Fig. 5): the per-vertex descent step, being mesh-resolution dependent, can't make progress against the TPE's high-frequency stiffness at fine mesh scales. The $H^{s}$ preconditioner in the next phase cures it.
03 — Reflection

On track, with one planned addition

We're on schedule relative to the proposal's Week 1 goal — "flowing a mesh toward a TPE minimum" — and the gradient is FD-checked everywhere it matters. Phase 1 consumed the full week rather than a few days, which was the right call: at $\alpha = 6$ the kernel has extreme stiffness at short range, and every sign error in the chain rule shows up as stalled descent rather than an exception. Catching those with a mandatory FD gate on every analytical derivative before anything depends on it is what made the Phase 1.8 demo work on the first try.

One addition to the plan. Our FD checks prove internal consistency (the analytical gradient matches the finite difference), not correctness (the kernel itself is right). A bug that flips a sign in both energy and gradient consistently is undetectable by FD. Before Phase 2 starts we will cross-validate against the reference CPU implementation (HenrikSchumacher/Repulsor) on a handful of meshes — running the same OBJs through both pipelines and comparing $\hat\Phi$ and $\nabla\hat\Phi$ to the Barnes-Hut tolerance at matching $\theta$. Repulsor is loaded as an external checkout (not vendored). This was not in the original proposal; it replaces "more genus-gallery meshes" as the Week 2 validation milestone.

Small slip. Higher-genus test meshes (genus $\geq 2$ closed surfaces) aren't cleanly available in any open repository we could find. We'll add a make_n_torus(g, …) procedural generator when Phase 2.13 (the canonical embedding gallery) needs it rather than committing a folder of hand-crafted OBJs now.

04 — Updated plan

Four weeks out

Week 1
Core engine. Mesh infrastructure, discrete TPE with BVH acceleration, gradient computation, FD-checked end to end. $L^2$ gradient flow on an ellipsoid reproduces the expected stall that motivates Phase 2.
Week 2
Optimization. Cotan Laplacian, fractional operators $B$ and $B_0$ with hierarchical compression for the $L^\sigma$ matvec, preconditioned GMRES, barycenter constraint via Schur complement, dynamic remeshing (split/collapse + Delaunay flip + tangential smoothing). Cross-validate TPE against Repulsor. By end of week: one Phase 2 canonical embedding (genus 0 or 1) converged.
Week 3
Gallery + GPU. Canonical-embedding gallery through genus 5 (sphere, torus, triangular prism, tetrahedral, and up). Start the CUDA port of the Barnes-Hut energy and gradient over the block-cluster tree — the single biggest CPU bottleneck per RS Fig. 33.
Week 4
Shells + hero demos. Membrane + bending shell energy, obstacle SDF barrier, adaptive near-field TPE quadrature. Target: ball-through-tube (RS Fig. 4 extrapolation). Cloth eversion (RS Fig. 1 interpolation) and the trust-region path-energy solver follow if time permits; GPU port of the $L^\sigma$ matvec in parallel.

Risk. Phase 2's hierarchical-compression step for the $L^\sigma$ matvec is the single trickiest piece of the whole pipeline — any bug silently tanks the preconditioned GMRES iteration count. We plan to gate it the same way we gated Phase 1: a $\theta = 0$ fast-vs-brute check to machine precision, and a $\theta = 0.5$ check below 1% relative error, before any downstream code uses it.
